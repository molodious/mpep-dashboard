Problem 13 — Isentropic Steam Expansion
Thermodynamics · Isentropic Process · Steam Tables
Steam with a quality of 90% expands isentropically from 350 psia to 120 psia. What is the change in enthalpy?
A. 40 Btu/lb
B. 80 Btu/lb ✓
C. 90 Btu/lb
D. 120 Btu/lb
Answer: B — 80 Btu/lb
1. Recognize that "isentropic" means constant entropy: s₁ = s₂. Entropy is the bridge linking State 1 to State 2. 2. Look up State 1 properties at 350 psia in the saturated steam tables (by pressure): h_f = 409.8, h_fg = 794.62, s_f = 0.6059, s_fg = 0.8914 3. Calculate h₁ and s₁ using quality x₁ = 0.90: h₁ = 409.8 + 0.9 × 794.62 = 1125 Btu/lbm s₁ = 0.6059 + 0.9 × 0.8914 = 1.408 Btu/(lbm·°R) 4. Set s₂ = s₁ = 1.408. Look up State 2 properties at 120 psia: h_f = 312.55, h_fg = 878.2, s_f = 0.4919, s_fg = 1.0965, s_g = s_f + s_fg = 1.588 5. Verify steam is still two-phase: s₂ = 1.408 < s_g = 1.588 ✓ 6. Solve for exit quality x₂: x₂ = (s₂ − s_f) / s_fg = (1.408 − 0.4919) / 1.0965 = 0.835 7. Calculate h₂: h₂ = 312.55 + 0.835 × 878.2 = 1046 Btu/lbm 8. Compute change in enthalpy: Δh = h₁ − h₂ = 1125 − 1046 = 79 Btu/lbm ≈ 80 Btu/lb → Answer B
Step 1 — Identify the Process and Key Relationship An isentropic process is one with no heat transfer and no irreversibilities — entropy is conserved: s₁ = s₂ This is the critical equation. Entropy at State 1 becomes the known quantity used to fully define State 2. Step 2 — Define State 1 (350 psia, x₁ = 0.90) Look up saturated steam table properties at 350 psia: h_f = 409.8 Btu/lbm h_fg = 794.62 Btu/lbm s_f = 0.6059 Btu/(lbm·°R) s_fg = 0.8914 Btu/(lbm·°R) Compute State 1 enthalpy and entropy using quality x₁ = 0.90: h₁ = h_f + x₁ · h_fg h₁ = 409.8 + (0.90)(794.62) h₁ = 409.8 + 715.2 h₁ = 1125 Btu/lbm s₁ = s_f + x₁ · s_fg s₁ = 0.6059 + (0.90)(0.8914) s₁ = 0.6059 + 0.8023 s₁ = 1.408 Btu/(lbm·°R) Step 3 — Set s₂ = s₁ and Look Up State 2 Properties (120 psia) Since the process is isentropic: s₂ = 1.408 Btu/(lbm·°R) Look up saturated steam table properties at 120 psia: h_f = 312.55 Btu/lbm h_fg = 878.2 Btu/lbm s_f = 0.4919 Btu/(lbm·°R) s_fg = 1.0965 Btu/(lbm·°R) s_g = s_f + s_fg = 0.4919 + 1.0965 = 1.5884 Btu/(lbm·°R) Step 4 — Verify the Steam Remains Two-Phase Check: s_f < s₂ < s_g ? 0.4919 < 1.408 < 1.5884 ✓ Yes — the steam is still in the two-phase (wet steam) region at State 2. We can use the quality equations. Step 5 — Find Exit Quality x₂ x₂ = (s₂ − s_f) / s_fg x₂ = (1.408 − 0.4919) / 1.0965 x₂ = 0.9161 / 1.0965 x₂ = 0.835 The steam expanded from 90% quality to 83.5% quality — it got slightly wetter as it expanded to lower pressure, which is physically reasonable. Step 6 — Calculate State 2 Enthalpy h₂ = h_f + x₂ · h_fg h₂ = 312.55 + (0.835)(878.2) h₂ = 312.55 + 733.3 h₂ = 1046 Btu/lbm Step 7 — Compute the Change in Enthalpy Δh = h₁ − h₂ = 1125 − 1046 = 79 Btu/lbm ≈ 80 Btu/lb → Answer B ✓ The steam lost 79 Btu/lbm of enthalpy as it expanded — this energy went to doing work on the surroundings. Why Not the Other Answers? • Answer A (40 Btu/lb): Approximately half the correct value. This could result from forgetting to multiply h_fg by quality (i.e., computing h₂ incorrectly), or from using only the s_f difference without dividing by s_fg. • Answer C (90 Btu/lb): Close to the right answer but overstates the drop. Might arise from using slightly different steam table values (e.g., rounding s₁ up) or miscalculating x₂ as slightly lower. • Answer D (120 Btu/lb): Could come from confusing h_fg at one of the pressures with Δh, or from computing a state property difference incorrectly (e.g., taking h_f₁ − h_f₂ = 409.8 − 312.55 ≈ 97 and rounding up). The correct approach always goes through entropy to find exit quality, then uses that quality to compute exit enthalpy.

Steam tables organized by pressure: Dan is explicit about using the saturated steam tables sorted by pressure (not temperature), since both states are given as pressures. He walks through each table lookup methodically — reading h_f, h_fg, s_f, and s_fg at each pressure before doing any arithmetic.

The two-phase check: Before computing exit quality, Dan verifies that s₂ lies between s_f and s_g at 120 psia. This is an important habit — if s₂ were greater than s_g, the steam would be superheated and you'd need superheated steam tables instead. Checking first prevents a systematic error.

State-by-state discipline: Dan emphasizes defining both states fully before computing differences. He writes out all six table values (h_f, h_fg, s_f, s_fg at both pressures) before solving for x₂ or h₂. This structured approach reduces errors and is directly exam-reproducible.

The calculation flow:
  • State 1: h₁ = 409.8 + 0.9 × 794.62 = 1125 Btu/lbm
  • State 1: s₁ = 0.6059 + 0.9 × 0.8914 = 1.408 Btu/(lbm·°R)
  • State 2 quality: x₂ = (1.408 − 0.4919) / 1.0965 = 0.835
  • State 2: h₂ = 312.55 + 0.835 × 878.2 = 1046 Btu/lbm
  • Δh = 1125 − 1046 = 79 ≈ 80 Btu/lb → Answer B

Key teaching moment: Dan points out that the exit quality (83.5%) is slightly lower than the inlet quality (90%). When wet steam expands to lower pressure isentropically, quality decreases — some vapor condenses. This is the physically expected result and serves as a built-in sanity check.
Key Takeaways
  • Isentropic = constant entropy. The equation s₁ = s₂ is not background info — it's the central tool that lets you find exit quality at the new pressure.
  • Always check the two-phase region. Before using quality equations at State 2, verify s_f < s₂ < s_g at the exit pressure. If s₂ > s_g, the steam is superheated and you need different tables.
  • Entropy first, enthalpy second. Never try to compute Δh directly. Find s₁ → set s₂ = s₁ → solve for x₂ → compute h₂ → then find Δh.
  • Use steam tables organized by pressure when the given conditions are pressures (as here). Table lookup order: 350 psia → read s_f, s_fg; 120 psia → read h_f, h_fg, s_f, s_fg.
  • Quality drops during isentropic expansion of wet steam — a sanity check. Going from 90% quality to 83.5% is physically expected and confirms you haven't made a sign error.
